[0:50 — ARCHITECTURE]
(show assets/workflow_diagram.png)
Let's walk through the architecture diagram to see how the system coordinates behind the scenes.
First, every traveler request or event trigger enters through the START node and immediately hits our Security Checkpoint.
Here, we check for prompt injection bypasses and scrub PII (such as passports and credit card numbers), routing any blocked events to a secure audit log.
Passed requests reach the main Orchestrator, which delegates tasks to 9 specialized sub-agents—ranging from Memory and Planner to Flight and Hotel tracking.
Finally, when booking changes are proposed, the workflow routes through our Human-in-the-Loop (HITL) Confirmation node to request user approval before writing to the database.

Let's see TripPilot-AI in action by running through a live simulation.
